Local Storage and Session Storage are the two main types of browser storages. They are very much important for the browser to provide better user experience. Though they works quiet similarly, yet there are some major differences between them.
The main difference is that local storage data does not expire automatically, whereas session storage data gets cleared when the user closes its browser window. As a result, if we close a tab or restart the browser the session storage will be cleared. But local storage will not be cleared unless it is deleted via JavaScript or manually.

The global scope is the outermost scope and it can be accessed from any other local or block scope. In a browser environment, the topmost scope of JavaScript file loaded using <script> tag is a global scope. A variable that is declared inside the global scope is called global variable. Global variable can be accessed from any place of the webpage's JavaScript.
The block scope is the scope of a particular code block.Block scope variables let or const declared in the block scope can only be accessible in that scope and can not be accessed in the global scope or anywhere else.
